📖 0. วิธีอ่านสเปกนี้

แต่ละกล่องคือ "ความสามารถหนึ่งอย่าง" ที่ HIS ต้องจัดให้ — อ่านจากบนลงล่างได้เลย ไม่ต้องมีพื้นฐานโปรแกรมมาก

READ Smart IPD ขออ่านจาก HIS WRITE Smart IPD ส่งกลับให้ HIS EVENT HIS ส่งเข้ามาเมื่อมีเหตุการณ์ MUST จำเป็น OPT เสริม (ทีหลังได้)

ช่องทางที่ HIS เลือกจัดได้: 1 = เปิด DB ให้อ่าน   2 = เปิด API ของ HIS   3 = HIS push เข้ามา  — เลือกแบบไหนก็ได้/ผสมได้

เรื่อง Path / Endpoint (เพิ่มใน v2.1)

แต่ละกล่องมี ตัวอย่าง path ของ API เช่น GET /his/patients/admitted — เป็นเพียงข้อเสนอ HIS จะตั้งชื่อ endpoint จริงเป็นอะไรก็ได้ ขอแค่มีความสามารถนั้น · Base URL สมมติ เช่น https://his.hospital.org/api/v1 · EVENT = HIS เป็นฝ่ายเรียกเข้า "webhook ของ Smart IPD"

🔑 1. รหัสอ้างอิงที่ต้องแนบมาทุกครั้ง

Smart IPD เก็บข้อมูล IPD ของตัวเอง แต่ต้อง "อ้างถึง" สิ่งของ HIS — ขอให้ส่งรหัสเหล่านี้เป็น "ค่าที่ HIS ใช้จริง" มาด้วย (Smart IPD จะเก็บไว้ตรง ๆ ไม่ดัดแปลง)

ฟิลด์คือตัวอย่าง
hnเลขประจำตัวผู้ป่วย (Hospital Number) — 1 คน 1 เลข ตลอดชีวิต6712345
anเลขการรับเข้านอน (Admission Number) — 1 ครั้งที่นอนAN680112
userIdรหัสบุคลากร (แพทย์/พยาบาล/เภสัชกร)D00123
drugCodeรหัสยาในบัญชีDRG0045
labCode / xrayCodeรหัสรายการแล็บ / เอกซเรย์LAB0102 / XR0033
wardCode / bedCodeรหัสวอร์ด / เตียงWD03 / BED12
orderIdรหัสคำสั่งที่ HIS สร้างหลังรับคำสั่ง (ส่งกลับมาให้ Smart IPD)ORD78901

🟢 12. ส่วนที่ Smart IPD ทำเอง (HIS ไม่ต้องจัด)

เพื่อความชัดเจน — รายการเหล่านี้ Smart IPD เก็บ/กำหนดเอง HIS ไม่ต้องเตรียม

รายการทำไมเป็นของ Smart IPD
Smart IPD เอกสารคลินิก IPD — บันทึกแพทย์/พยาบาล, แผนการพยาบาล, คำสั่ง (ต้นฉบับ), eMAR, สรุปจำหน่าย, ทบทวนยา, รับผู้ป่วยใหม่เนื้อหาเกิด/ใช้เฉพาะใน IPD
Smart IPD Master น้ำเข้า-ออก (Intake/Output) + คลังแม่แบบงานพยาบาล (FAIE)config เฉพาะงาน IPD
Smart IPD ระบบล็อกอิน/สิทธิ์, ผู้ช่วย AI, ป้ายกำกับผู้ป่วยฟีเจอร์เฉพาะ Smart IPD

สรุป: HIS จัดเฉพาะ "ข้อมูลกลางของโรงพยาบาล" (ข้อ 2–10) — ส่วน "เนื้อหาการดูแลผู้ป่วยใน" Smart IPD ดูแลเอง

🔒 13. ความปลอดภัย & Checklist

ความปลอดภัย / สิ่งที่ต้องตกลง

  • การเข้าถึง: DB → บัญชี อ่านอย่างเดียว · API → ระบุวิธียืนยันตัวตน (OAuth2 / API key / certificate)
  • เครือข่าย/ความลับ: เปิด firewall เฉพาะที่จำเป็น · รับส่งผ่าน HTTPS/TLS
  • กันส่งซ้ำ: SubmitOrder ต้อง idempotent — Smart IPD อ้าง orderId ที่ได้กลับ
  • ระบบทดสอบ: มีชุด Test/UAT + ข้อมูลตัวอย่าง ก่อนเปิด Production
  • เมื่อ HIS ล่ม: อ่านใช้ cache ได้ · เขียนเข้าคิวรอ retry

✅ Checklist ขั้นต่ำเพื่อเริ่มใช้งาน (Go-Live)

  • GetAdmittedPatients + PatientEvents — รายชื่อในวอร์ด + เหตุการณ์ ADT
  • GetPatientInfo / AdmissionCard — ข้อมูลผู้ป่วย + แพ้ยา
  • นัดหมาย — อ่านปฏิทิน + สร้าง/แก้/ยกเลิก/เลื่อน (ถ้าใช้ฟีเจอร์นัด)
  • Master: วอร์ด/เตียง, ห้อง, urgent, general order
  • Master: ICD + diagnosis types + discharge status/type + organs
  • บัญชียา — ค้นยา + timing/route/form/unit + ราคา
  • แล็บ + เอกซเรย์ catalog — รายการ + group/modality
  • บุคลากร — แพทย์/พยาบาล/เภสัชกร
  • ผลแล็บ + ผลเอกซเรย์ + PACS
  • สัญญาณชีพ — อ่าน + เขียน (ถ้า HIS เป็นเจ้าของ)
  • SubmitOrder — รับคำสั่ง + ตอบ orderId
  • UpdateOrder + OrderStatusFeedback — แก้/ยกเลิก + แจ้งสถานะกลับ
  • รหัสอ้างอิง (HN/AN/userId/itemCode) + Auth + ระบบทดสอบ
